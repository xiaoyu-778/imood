## 系统概览

iMood v2 是一个基于 Node.js 的全栈应用，采用经典的 **MVC 架构** 模式：

```
┌─────────────────────────────────────────────────────────────┐
│                        客户端 (Browser)                       │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐       │
│  │  UI 渲染层    │  │ 音频处理    │  │ 3D 渲染     │       │
│  │  (HTML/CSS)  │  │ (Web Audio) │  │ (Three.js)  │       │
│  └──────────────┘  └──────────────┘  └──────────────┘       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼ HTTP/WebSocket
┌─────────────────────────────────────────────────────────────┐
│                      服务端 (Node.js)                        │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐       │
│  │ 路由层       │  │ 业务逻辑    │  │ 数据访问    │       │
│  │ (Express)    │  │ (Services) │  │ (JSON DB)   │       │
│  └──────────────┘  └──────────────┘  └──────────────┘       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼ REST/SSE
┌─────────────────────────────────────────────────────────────┐
│                      外部服务 (AI API)                       │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐       │
│  │ 文本对话    │  │ 语音合成    │  │ 图片理解    │       │
│  │ (Chat API)  │  │ (TTS API)   │  │ (Vision API)│       │
│  └──────────────┘  └──────────────┘  └──────────────┘       │
└─────────────────────────────────────────────────────────────┘
```

```javascript
class AIService {
  // 文本对话 - SSE 流式响应
  async chatStream(message, sessionId, res) {}
  
  // 语音对话
  async chatAudio(audioFile, sessionId) {}
  
  // 图片对话
  async chatImage(imageFile, sessionId) {}
  
  // 生成日记元数据

}
```

**设计要点：**
- 使用 SSE (Server-Sent Events) 实现流式响应
- 统一错误处理和重试机制
- 支持多模型切换

## 设计决策

### 1. 为什么选择 JSON 文件存储？

**决策：** 使用 JSON 文件而非数据库

**理由：**
- ✅ 部署简单，无需配置数据库
- ✅ 数据可移植，易于备份
- ✅ 读写频率低，性能足够
- ✅ 个人应用，数据量小

**权衡：**
- ❌ 不支持并发写入
- ❌ 查询能力有限
- ❌ 无事务支持

**未来演进：** 可迁移到 SQLite 或 PostgreSQL

### 2. 为什么使用原生 JavaScript？

**决策：** 不使用 React/Vue 等框架

**理由：**
- ✅ 减少依赖，降低复杂度
- ✅ 加载速度快
- ✅ 完全控制渲染逻辑
- ✅ 适合单页面应用

**权衡：**
- ❌ 代码组织需要更多规范
- ❌ 组件复用性较低

### 3. 为什么使用 SSE 而非 WebSocket？

**决策：** AI 流式响应使用 SSE

**理由：**
- ✅ 基于 HTTP，易于调试
- ✅ 自动重连机制
- ✅ 单向通信足够（服务器→客户端）
- ✅ 兼容性好

**权衡：**
- ❌ 不支持双向实时通信
- ❌ 连接数限制

### 4. 为什么使用 OpenAI SDK？

**决策：** 统一使用 OpenAI 兼容格式

**理由：**
- ✅ 行业标准，文档丰富
- ✅ 多厂商兼容（阿里云、百度等）
- ✅ 易于切换模型
- ✅ 功能完整（流式、函数调用等）

## 扩展性

### 水平扩展

当前架构为单机部署，如需扩展：

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Nginx     │────►│  iMood #1   │     │  SQLite/    │
│  (负载均衡)  │     │  (Node.js)  │────►│  PostgreSQL │
│             │────►│  iMood #2   │     │             │
└─────────────┘     │  (Node.js)  │     └─────────────┘
                    └─────────────┘
```

**需要修改：**
1. 替换 JSON 文件为数据库
2. 使用 Redis 存储会话状态
3. 文件存储使用对象存储（OSS/S3）

### 功能扩展

| 功能 | 实现方案 | 难度 |
|------|----------|------|
| 用户系统 | JWT + 数据库 | ⭐⭐ |
| 多语言支持 | i18n 库 | ⭐ |
| 数据导出 | PDF/CSV 生成 | ⭐⭐ |
| 情绪统计 | 图表库 (Chart.js) | ⭐ |
| 语音唤醒 | Web Speech API | ⭐⭐⭐ |
